Step 1: SSH Keys

  • Make key on your machine ssh--keygen -t rsa
  • DSA or RSA? See Check out StackOverflow
  • Key is in a .pub file

Step 2: Send to the Cluster

  • Use Cyberduck (Mac) or WinSCP (Windows) or rsync (in Terminal) to send to cluster
  • Go on cluster, copy to .ssh folder.
  • cat file.pub >> authorized_keys, (>> means append)
  • Do the SAME thing on the cluster, creating jhpce_file.pub, and then do cat jhpce_file.pub >> authorized_keys
  • This will allow for passwordless node login

Step 3: Login into node from one command

  • Typing Passwords is BORING and inefficient
  • Check out Leo's post for qr: http://lcolladotor.github.io/2013/12/11/quick-cluster-login-to-interactive-session/ c In .bash_profile (defining environment variables and aliases):

My addition: qrgig

Use qrgig 5 6:

qrgig(){
    x="$1"
    y="$2"
    y="${y}G"
    x="${x}G"
    # echo "${x}"
    # echo "${y}"
    ell="-l mem_free=$x,h_vmem=$y,$3"
    echo "qrsh requests were: $ell"
    cmd="source /etc/profile; echo \"cd \$PWD\" > ~/.bash_pwd; history -w; qrsh $ell"
    # echo "$cmd"
    justenigma -t $cmd
}

Port Forwarding

This link allows you to forward your code to a local machine: http://erniemiller.org/2011/12/12/textmate-2-rmate-awesome/

Then put this in your .bash_profile on the cluster, changing MYPORT to a unique number: alias subl="rmate -p MYPORT"

Then put this in your local .bash_profile:

alias enigma="ssh -Y -X -R MYPORT:localhost:52698 ${jenig}"

Go onto JHPCE, then type subl FILE
